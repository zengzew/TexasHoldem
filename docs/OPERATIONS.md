# 运维与回归清单（v2.3.1）

## 1. 每次发布前

1. `npm test -- --run` 通过
2. `npm run build` 通过
3. 核心流程冒烟：
- 登录/注册
- 创建房间
- 加入房间
- 录入买入与最终积分
- 玩家“确定”按钮 loading 与防重复提交流程
- 结算与转账建议
- 房主管理玩家（添加 / 移除）
- 历史记录分页与展开
- 个人看板累计总览与近 10 局趋势
- 积分榜切换排序
- 积分榜展开详情高级指标（最大单局盈利/亏损）
- 积分榜日期范围筛选（快捷区间 + 自定义开始/结束日期）
- 页面刷新后自动恢复当前房间

## 2. 发布后 5 分钟检查

1. 首屏是否正常渲染（桌面 + 移动端）
2. Supabase Realtime 是否生效（双端进同一房间）
3. 结算后历史记录、积分榜是否同步刷新
4. 账户设置（昵称/密码）是否可保存
5. 房主添加玩家是否成功（无 RLS 拒绝）

## 3. 浏览器兼容检查

- Safari（iPhone）
- Edge（Windows）
- Chrome（桌面）

关注点：
- panel 高度与分页器位置
- tab 栏样式是否坍塌
- 输入框与按钮对齐

## 4. 常见应急处理

1. 页面白屏
- 检查 Vercel env 是否缺失
- 浏览器控制台检查 `VITE_SUPABASE_*`

2. 数据不同步
- 检查 Realtime 开关表是否正确
- 检查 RLS 是否阻断订阅读取

3. 登录异常
- 检查 `nickname_exists` 与 `resolve_login_emails` 函数是否存在
- 确认 `profiles` 中昵称与 `auth.users` 关联正常
