# Changelog

## v2.1.5 (2026-03-09)

- 当前房间玩家管理区交互优化
  - 移除“房主管理玩家”标题文案
  - 房主添加玩家区仅在“全部玩家”视图展示
  - 添加玩家支持左匹配模糊搜索（昵称前缀匹配 + 候选下拉）
- 数字反馈动画
  - 玩家卡片“累计总买入”新增数值变化动画
  - 当前房间汇总“总买入”新增数值变化动画

## v2.1.4 (2026-03-08)

- 房主玩家管理
  - 新增“房主管理玩家”面板，支持按昵称添加玩家
  - 新增移除玩家能力（仅房主，且房主自身不可移除）
  - `room_players` RLS 策略升级为 `own_or_owner`（insert/update/delete）
- 买入确认防重复提交
  - 玩家点击“确定”后按钮进入 loading
  - 增加每玩家 in-flight 防抖，避免重复写入
- 积分榜“场均盈利”逻辑修正
  - 改为按“场均盈利金额（RMB）”排序与展示
- 刷新保留当前房间
  - 新增本地持久化房间状态（按用户隔离 key）
  - 页面刷新后自动尝试恢复房间，失效房间自动清理缓存并提示
- 工程修复
  - 清理 `App.jsx` 孤立 mock 代码块导致的构建错误
  - 新增 `roomState` 工具测试，补充场均金额排序测试

## v2.1.3 (2026-03-03)

- 全站时间处理统一为北京时间（`Asia/Shanghai`）
  - 顶部日期范围、历史记录时间展示统一北京时间
  - 默认房间号日期改为北京时间
  - 积分榜日期筛选（预设与自定义）按北京时间日界限计算
- DatePicker 交互优化
  - 开始/结束改为左右并排
  - 日期输入显示统一为 `YYYY-MM-DD`
  - 顶部日期范围字体放大一档
- 刷新图标交互优化
  - 点击刷新后强制执行单次 `360°` 旋转动画，不依赖 loading 时长
- 历史记录金额可读性优化
  - 玩家结果中 RMB 金额按盈亏红绿区分（盈利绿、亏损红）

## v2.1.2 (2026-03-03)

- 未结算房间“刷新”按钮增加请求中旋转动画，明确反馈刷新状态
- Header 面板上下留白进一步收紧，整体高度小幅降低并保持布局和谐

## v2.1.0 (2026-03-02)
